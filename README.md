# Автоматическое сопоставление участников оценки с сотрудниками

## Что это и зачем

В рамках задачи по автоматизации отчётов данные об одном сотруднике приходят из
разных источников (360, PIF, ЭРА, SkillCode) в разных форматах. Часто в записи
нет email или отчества, а среди сотрудников встречаются однофамильцы. Чтобы
собрать единую карточку сотрудника, нужно надёжно определять, относятся ли две
записи к одному человеку.

Это решение - **обучаемая модель**, которая по паре "запись из отчёта - сотрудник"
оценивает вероятность того, что это один и тот же человек, и принимает решение:
связать автоматически, отправить на ручную проверку или не связывать.

## Что сделано

1. **Признаки сходства пары.** По каждой паре считается 10 признаков: несколько мер
   сходства ФИО (устойчивых к перестановке слов, опечаткам и пропущенному отчеству),
   совпадение email, совпадение логина с email, совпадение руководителя.

2. **Подготовка обучающих данных.** Поскольку готовой размеченной выборки нет,
   данные смоделированы с реалистичным шумом (опечатки, "ё/е", пропуск отчества,
   отсутствие email). Главное - в обучение специально включены **трудные
   отрицательные примеры**: однофамильцы и похожие ФИО. Без них модель научилась бы
   отличать только очевидно разных людей и ошибалась бы там, где это важнее всего.

3. **Обучение и сравнение моделей.** Обучены логистическая регрессия и градиентный
   бустинг, выбрана лучшая по качеству.

4. **Калибровка вероятностей.** Вероятности откалиброваны, чтобы значение "0.9"
   действительно означало ~90% уверенности - это нужно для осмысленного порога.

5. **Порог под бизнес-логику.** Порог автоматической связки выбран так, чтобы почти
   не допускать ошибочных слияний (точность >= 0.99). Всё, в чём модель не уверена,
   уходит на ручную проверку - потому что ошибочно объединить двух разных людей
   дороже, чем потратить время на проверку.

## Результаты

| Модель | ROC-AUC | PR-AUC | F1 |
|---|---|---|---|
| Логистическая регрессия | 0.95 | 0.90 | 0.83 |
| **Градиентный бустинг** (выбрана) | **0.996** | **0.99** | **0.96** |

- При пороге автоматической связки точность ≈ 0.99 - почти все автоматические
  сопоставления верны.
- Самый важный признак - совпадение руководителя: именно он позволяет различать
  однофамильцев. Далее идут меры сходства ФИО и совпадение логина.
- На примере двух однофамильцев (Абызов Е.И. и Абызов Е.О.) без отчества и
  руководителя модель не выбирает наугад, а отправляет случай на ручную проверку.

Графики (ROC, точность-полнота, калибровка, важность признаков) - в папке `figures/`.

## Состав

```
data -> dataset.py        подготовка обучающих пар (с трудными примерами)
       features.py       вычисление признаков сходства пары
обуч. -> train.py         обучение, сравнение, калибровка, графики, сохранение модели
примен -> predict.py      применение модели к новым парам
модель -> models/         сохранённая модель + карточка модели (метрики, порог, признаки)
графики -> figures/       ROC, precision-recall, калибровка, важность признаков
разбор -> ml_matching_analysis.ipynb   пошаговый ноутбук со всеми результатами
```

## Как запустить

```bash
pip install rapidfuzz scikit-learn matplotlib joblib numpy pandas
python3 train.py        # обучение, метрики и графики
python3 predict.py      # пример применения к новым парам
```

Полный разбор с пояснениями и графиками - в `ml_matching_analysis.ipynb`.

## Применение и развитие

Сейчас модель обучена на смоделированных данных. На реальных размеченных
сопоставлениях (которые накапливаются по мере ручной проверки) её можно дообучить
без изменения кода - признаки и логика остаются те же. Также возможно добавить
сравнение по руководителю и подразделению как дополнительные сигналы, когда они
доступны в источниках.
